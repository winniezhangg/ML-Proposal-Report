Credit Card Fraud Detection

1. Introduction / Background

Credit card fraud is a significant issue in the financial sector, leading to substantial financial losses. Fraudulent activities, such as card cloning, identity theft, and unauthorized transactions are common challenges for credit card companies. To combat this, many institutions use machine learning to detect unusual transaction patterns and flag potential fraud.

Past Cases / Literature Review

  • Safa and Ganga: They created a model that heavily utilized logistic regression to analyze distorted credit card data. The logistic regression method had a 99.07% accuracy [2].

Dataset Description

We plan to use publicly available datasets like the Kaggle Credit Card Fraud Detection Dataset, which contains real credit card transactions with fraudulent transactions. This dataset contains transactions from September 2013 by Europeans. This data is spanned over 2 days, where there are 492 frauds out of 284,807 transactions [3]. There are only numerical input variables such as time between the first transaction and current transaction.

Dataset Link: Kaggle Credit Card Fraud Dataset

2. Problem Definition

Problem

The challenge is to detect fraudulent transactions using both supervised methods (classification) and unsupervised methods (anomaly detection) to identify patterns that indicate fraud.

Motivation

The motivation behind creating a machine learning model to detect credit card fraud stems from the desire to protect consumers and financial institutions from significant financial losses. As online transactions and digital payments increase, so does the risk of fraud. This offers a more adaptive approach, learning from patterns in transactional data to detect suspicious activity in real-time, reducing false positives, and enhancing security without compromising user experience.

3. Methods

Preprocessing

  • Data Cleaning: Check for missing values and adjust the data accordingly, identify and remove any outliers that may be skewing the data.
  • Scaling: Normalization to ensure everything is on the same scale.
  • Converting Categorical Values: Convert categorical features into numerical columns, make sure to label numerical features where order matters.

Machine Learning Models

  • Supervised Learning: Logistic regression, Random Forest, or Gradient Boosting Classifiers for detecting fraud based on labeled data.
  • Unsupervised Learning: Use clustering algorithms like DBSCAN or Isolation Forest for anomaly detection, assuming fraud transactions are rare events.

4. Results and Discussion

Metrics

  • Accuracy & Precision: The total amount of transactions that are correctly classified as fraudulent/legitimate should be >80%.
  • Recall: The percent of fraudulent transactions the model is actually able to identify greater than >80%.
  • F1-Score: A metric that measures the models accuracy by utilizing precision and recall and >80%.
  • False Positive Rate (FPR): The percent of falsely classified transactions as fraudulent is less than <1%.
  • AUC-ROC Curve: A metric to measure the model's ability to identify fraud versus non-fraud should be greater than >0.7.

Project Goals

Improve fraud detection rate while minimizing false positives, hold people accountable ethically/morally for their actions, and promote the use of credit cards as models like these tend to make their use more safe and sustainable for consumers.

Expected Results

The expected results of a machine learning model for detecting credit card fraud should include high accuracy in identifying fraudulent transactions while minimizing false positives. The model should be able to flag suspicious activity with precision and effectively distinguish between legitimate and fraudulent transactions based on learned patterns. Additionally, it should adapt over time as new fraud techniques emerge, providing real-time detection and contributing to overall improved security for both users and financial institutions.
